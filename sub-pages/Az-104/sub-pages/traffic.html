Traffic Management & Load Balancing

Distribute incoming network traffic across multiple servers using Azure Load Balancer to ensure high availability and reliability for your applications.

Concept Refresher: Azure Load Balancer operates at Layer 4 of the OSI model (TCP/UDP). It distributes inbound flows that arrive at the load balancer's front end to backend pool instances. For HTTP/HTTPS specific Layer 7 load balancing (like URL-based routing), you would use an Application Gateway instead.

Exercise 1: Create a Public Load Balancer

First, we will provision the Load Balancer resource and assign it a Public IP address so it can receive traffic from the internet.

  1. Sign in to the Azure portal.
  2. In the search bar, type Load balancers and select it. Click + Create.
  3. Basics Tab: Select your Subscription and Resource Group. Name the Load Balancer az104-lb.
  4. Set the Type to Public and the SKU to Standard.
  5. Frontend IP configuration Tab: Click + Add a frontend IP configuration.
  6. Name it LoadBalancerFrontEnd. Create a new Public IP address named lb-pip. Click Add.
  7. Click Review + create, then click Create.

Exercise 2: Configure Backend Pool and Health Probe

The load balancer needs to know *where* to send the traffic (Backend Pool) and it needs a way to check if those servers are actually online and healthy (Health Probe).

  1. Once deployed, open your az104-lb resource.
  2. On the left menu, select Backend pools and click + Add. Name it Backend-Pool-1. Select your VNet and add any existing VMs you want to balance traffic across. Click Add.
  3. Next, select Health probes on the left menu and click + Add.
  4. Name the probe HTTP-Probe. Set the Protocol to HTTP and Port to 80.
  5. Leave the Interval at 5 seconds. This means Azure will check your VMs every 5 seconds to ensure port 80 responds. Click Add.

Exercise 3: Create a Load Balancing Rule

Finally, we tie the Frontend IP, the Backend Pool, and the Health Probe together with a rule dictating how the traffic should flow.

  1. Select Load balancing rules on the left menu and click + Add.
  2. Name the rule HTTP-Rule.
  3. Select your Frontend IP address (LoadBalancerFrontEnd) and Backend pool (Backend-Pool-1).
  4. Ensure Protocol is set to TCP and Port is set to 80.
  5. Select the HTTP-Probe you created in Exercise 2.
  6. Click Add.
Verification: Navigate to the Overview page of your Load Balancer and copy the Public IP address. Paste it into your browser. The load balancer will receive your request and seamlessly forward it to one of your healthy Backend VMs!
