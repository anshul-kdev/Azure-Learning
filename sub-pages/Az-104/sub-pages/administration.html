Azure Administration Tools

Move beyond the graphical portal. Learn to automate and manage Azure resources using command-line tools like Azure CLI, PowerShell, and Infrastructure as Code (ARM Templates).

Concept Refresher: While the Azure Portal is great for learning, real-world cloud administration relies heavily on automation. Azure Cloud Shell provides a browser-based, pre-authenticated terminal so you don't have to install local tools. ARM Templates (and Bicep) allow you to deploy resources declaratively using JSON files.

Exercise 1: Deploy Resources using Azure CLI

We will launch the Azure Cloud Shell and use the Azure Command-Line Interface (Bash) to quickly create a Resource Group without clicking through menus.

  1. Sign in to the Azure portal.
  2. Click the Cloud Shell icon (it looks like >_) located in the top navigation bar next to the search box.
  3. If prompted to mount storage, click Create storage. This provides a 5GB file share to store your scripts.
  4. When the shell opens at the bottom of your screen, select Bash from the environment dropdown menu in the top-left corner of the shell pane.
  5. To create a new Resource Group, type the following command and press Enter:
    az group create --name az104-cli-rg --location eastus
Verification: The terminal will immediately output a JSON block indicating "ProvisioningState": "Succeeded". You can now search for "Resource groups" in the portal UI and find your newly created group!

Exercise 2: Deploy Resources using Azure PowerShell

Azure PowerShell uses the 'Az' module and follows the standard Verb-Noun naming convention. Let's create another Resource Group, this time using PowerShell syntax.

  1. In the same Cloud Shell pane at the bottom of the portal, click the environment dropdown (which currently says "Bash").
  2. Switch the environment to PowerShell and click Confirm if prompted.
  3. Wait for the blue PowerShell prompt to load.
  4. To create a new Resource Group, type the following command and press Enter:
    New-AzResourceGroup -Name "az104-ps-rg" -Location "EastUS"
Verification: PowerShell will output a formatted table showing the ResourceGroupName, Location, and a ProvisioningState of Succeeded.

Exercise 3: Deploy via ARM Template

Infrastructure as Code (IaC) ensures repeatable and identical deployments. Azure Resource Manager (ARM) templates are JSON files that define the resources you want to deploy.

  1. Close the Cloud Shell by clicking the 'X' in the corner of its pane.
  2. In the top search bar of the Azure portal, type Deploy a custom template and select it.
  3. Click the link that says Build your own template in the editor.
  4. Delete any existing code in the editor and paste the following simple JSON template that creates a Virtual Network:
    5. {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Network/virtualNetworks",
      "apiVersion": "2021-02-01",
      "name": "az104-arm-vnet",
      "location": "[resourceGroup().location]",
      "properties": {
        "addressSpace": {
          "addressPrefixes": [
            "10.5.0.0/16"
          ]
        }
      }
    }
  ]
}
  5. Click Save at the top.
  6. Select one of the Resource Groups you created in the previous exercises, leave the region as is, and click Review + create followed by Create.
Verification: Once deployment completes, navigate to your resource group. You will see a fully configured Virtual Network named az104-arm-vnet built entirely from your JSON code!
