Infrastructure as Code (ARM Templates)

Automate the deployment of Azure resources using declarative JSON templates to ensure your infrastructure is consistent, repeatable, and version-controlled.

Concept Refresher: Azure Resource Manager (ARM) templates are written in JSON. They are declarative, meaning you state what you want the final environment to look like, and Azure figures out how to make it happen. Deployments are idempotent—if you deploy the same template twice, Azure will only make changes if the code has been updated; it won't duplicate the resources.

Exercise 1: Deploy a Quickstart Template

Microsoft maintains a massive library of pre-built ARM templates on GitHub. We will use one to rapidly deploy a standard resource.

  1. Sign in to the Azure portal.
  2. In the global search bar, type Deploy a custom template and select it.
  3. On the Custom deployment page, look under the "Template" dropdown. This connects directly to Azure Quickstart Templates.
  4. Click the dropdown and type/select 101-storage-account-create.
  5. Once selected, the portal automatically loads the required parameters. Select your Subscription and Resource Group.
  6. Notice that the template prompts you for specific values defined in its code (like Storage Account Type). Leave the defaults or adjust as desired.
  7. Click Review + create, then click Create.

Exercise 2: Export an ARM Template from an Existing Resource

What if you built a complex Virtual Machine manually in the portal, and now your boss wants you to deploy 50 exact copies of it? You don't have to write the code from scratch; you can reverse-engineer it using the Export tool.

  1. In the Azure portal, navigate to any Resource group you previously created (for example, the one holding your Virtual Network or Storage Account).
  2. On the left-hand menu of the Resource Group, scroll down to the Automation section and click Export template.
  3. Azure will generate the underlying JSON code that represents the current state of every resource inside that group.
  4. Click Download to save the .zip file to your local computer.
Observation: If you open the downloaded zip file, you will find two files: template.json (the resource definitions) and parameters.json (the specific values like names and regions). You can check these into Git, modify them, and deploy them again to recreate the exact environment!

Exercise 3: Edit and Deploy a Custom Template

Let's use the portal's built-in editor to manually modify a JSON template and pass parameters to it.

  1. In the top search bar, type Deploy a custom template and select it again.
  2. Click the link that says Build your own template in the editor.
  3. Delete any existing code in the editor and paste the following JSON:
    4. {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "vnetName": {
      "type": "string",
      "defaultValue": "az104-custom-vnet",
      "metadata": {
        "description": "Name of the virtual network"
      }
    }
  },
  "resources": [
    {
      "type": "Microsoft.Network/virtualNetworks",
      "apiVersion": "2021-02-01",
      "name": "[parameters('vnetName')]",
      "location": "[resourceGroup().location]",
      "properties": {
        "addressSpace": {
          "addressPrefixes": [
            "10.7.0.0/16"
          ]
        }
      }
    }
  ]
}
  4. Notice the parameters block. We defined a variable called vnetName.
  5. Click Save at the top.
  6. You are returned to the deployment screen. Notice how the Azure Portal read your JSON and automatically created a text box for "Vnet Name" with your default value filled in!
  7. Select a Resource Group, click Review + create, and deploy.
