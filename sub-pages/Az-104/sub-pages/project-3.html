Capstone Project 3

Automated Corporate Data Archive

Combine Azure Storage, Blob Lifecycle Management, Shared Access Signatures (SAS), and Azure Backup to build a compliant, cost-optimized data vault.

The Scenario: Your organization generates thousands of internal compliance reports every month. You need to provide a central cloud file share for employees to drop these files. To save money, files untouched for 30 days must be moved to cheaper storage automatically. Finally, external auditors need temporary read-only access to specific folders without needing a company login.

Phase 1: Storage Provisioning

First, we will establish the foundational storage account, including containers for raw blob data and SMB file shares for legacy file access.

  1. Sign in to the Azure Portal and create a new Resource Group named Contoso-Archive-RG.
  2. Create a Storage Account named contosoarchive[yourinitials] (must be globally unique, lowercase, no spaces).
  3. Select Standard performance and Geo-redundant storage (GRS) to protect against regional outages. Click Review + Create.
  4. Once deployed, navigate to the storage account. Under Data storage, click Containers. Create a container named compliance-reports.
  5. Under Data storage, click File shares. Create a file share named employee-dropzone with a quota of 100 GiB.

Phase 2: Lifecycle Automation

Hot storage is expensive. We don't want to pay premium rates for reports that no one looks at after the first month. We will automate the tiering process.

  1. Inside your Storage Account menu, scroll down to the Data management section and select Lifecycle management.
  2. Click + Add a rule and name it ArchiveOldReports. Leave the scope applied to "Limit blobs with filters" or apply to all blobs in the container.
  3. Under the Base blobs tab, configure the following IF/THEN logic:
    • IF base blobs were last modified more than 30 days ago... THEN move to Cool storage.
    • Click "Add an action". IF base blobs were last modified more than 90 days ago... THEN move to Archive storage.
  4. Click Add to save the rule. Azure will now automatically scan and tier your data every 24 hours.

Phase 3: Secure External Access (SAS)

An auditor has requested access to the compliance-reports container. They do not have an Azure AD account, and you cannot make the container public.

  1. Navigate to your compliance-reports container. Upload a dummy text file to act as a report.
  2. Instead of granting access to the whole account, click Shared access tokens (or Generate SAS) from the container menu.
  3. Configure the SAS token parameters:
    • Permissions: Check Read and List only. (Uncheck Write/Delete).
    • Start and expiry date/time: Set the expiry time for exactly 8 hours from now.
    • Allowed IP addresses: (Optional) Restrict it to the auditor's corporate IP.
    • Allowed protocols: HTTPS only.
  4. Click Generate SAS token and URL. Copy the generated Blob SAS URL and test it in a private browser window.

Phase 4: Azure File Share Backup

Employees are dropping critical files into the employee-dropzone SMB share. We must protect against accidental deletion or ransomware.

  1. In the global search bar, search for and create a Recovery Services vault named Contoso-Backup-Vault in the same region as your Storage Account.
  2. Once deployed, open the vault. Under the Getting Started section, click + Backup.
  3. For Where is your workload running?, select Azure.
  4. For What do you want to backup?, select Azure FileShare and click Backup.
  5. Select the contosoarchive[yourinitials] storage account you created in Phase 1.
  6. Select the employee-dropzone file share.
  7. Choose the default Daily Backup policy (which retains data for 30 days) and click Enable Backup.
Project Complete!
You have successfully built an automated, cost-optimized, and highly secure data archive. You utilized Storage Accounts for SMB/REST access, Lifecycle Management to crush cloud costs, SAS Tokens for zero-trust external sharing, and Recovery Services Vaults to guarantee business continuity!
