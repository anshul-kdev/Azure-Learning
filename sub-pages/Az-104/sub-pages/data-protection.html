Data Protection & Backup

Ensure business continuity by implementing Azure Backup for Virtual Machines using a Recovery Services Vault.

Concept Refresher: A Recovery Services Vault is a storage entity in Azure that houses data such as backup copies, recovery points, and backup policies. Crucial Note: A vault can only protect resources that are in the same region as the vault itself.

Exercise 1: Create a Recovery Services Vault

First, we must provision the vault where the backup data and retention policies will be securely stored.

  1. Sign in to the Azure portal.
  2. In the search bar, type Recovery Services vaults and select it. Click + Create.
  3. Select your Subscription and Resource Group. Name the vault AZ104-Backup-Vault.
  4. Select the exact same Region where the Virtual Machine you want to back up currently resides.
  5. Click Review + create, wait for validation, and click Create.

Exercise 2: Configure a Virtual Machine Backup

We will now attach a backup policy to an existing Virtual Machine to ensure it is backed up automatically every day.

  1. Once deployed, open your AZ104-Backup-Vault.
  2. On the left menu under the Getting started section, click on Backup.
  3. For Where is your workload running?, select Azure.
  4. For What do you want to backup?, select Virtual Machine, then click Backup.
  5. Under the Backup policy dropdown, choose the DefaultPolicy (which runs daily at a specified time and retains data for 30 days).
  6. Under Virtual Machines, click Add. Check the box next to one of your existing VMs and click OK.
  7. Click Enable backup.

Exercise 3: Trigger an On-Demand Backup

Automated backups are great, but if you are about to perform a major software update on a server, you should always take an immediate manual backup just in case.

  1. Still inside your vault, go to the left menu under Protected items and click Backup items.
  2. Click on Azure Virtual Machine.
  3. Click on the Virtual Machine you just enrolled in Exercise 2.
  4. At the top menu of the backup item dashboard, click Backup now.
  5. Accept the default retention date for this manual backup and click OK.
Verification: Once the backup job completes, the Restore VM and File Recovery buttons will light up. You can use these to either roll back the entire Virtual Machine or mount a recovery point to fetch individual lost files!
