Virtual Networks and NSGs

Design and configure a secure foundational network architecture in Azure using Virtual Networks, custom Subnets, Network Security Groups, and Azure DNS.

Concept Refresher: A Virtual Network (VNet) acts as a boundary for your resources. It represents a private address space (e.g., 10.0.0.0/16). You segment this large block into smaller pieces called Subnets (e.g., 10.0.1.0/24) to organize resources. To protect these resources, you apply a Network Security Group (NSG) to the subnet, which acts like a firewall filtering incoming and outgoing traffic.

Exercise 1: Create a Custom Virtual Network

We will create a foundational network with an explicit IP address space and split it into two subnets: one for front-end web servers and one for back-end databases.

  1. Sign in to the Azure portal.
  2. In the top search bar, type Virtual networks and select it.
  3. Click + Create.
  4. Basics Tab: Select your Subscription and Resource Group. Name the VNet Core-VNet. Choose a Region (e.g., East US).
  5. IP Addresses Tab: By default, Azure assigns an IPv4 address space like 10.0.0.0/16. Keep this space.
  6. Under Subnets, you will see a `default` subnet (10.0.0.0/24). Click on it to edit it, and rename it to Frontend. Save.
  7. Click + Add subnet. Name it Backend, and set the Subnet address range to 10.0.1.0/24. Click Add.
  8. Click Review + create, then click Create.

Exercise 2: Create a Network Security Group (NSG)

By default, subnets allow traffic to flow freely within the VNet. To secure the `Frontend` subnet so it only accepts web traffic from the internet, we need an NSG.

  1. In the Azure portal search bar, type Network security groups and select it.
  2. Click + Create.
  3. Select the same Resource Group and Region as your VNet. Name the NSG Frontend-NSG. Click Review + create and then Create.
  4. Once deployed, go to the resource. On the left menu under Settings, click Inbound security rules.
  5. Click + Add to create a new rule.
  6. Configure the rule as follows:
    • Source: Any
    • Source port ranges: *
    • Destination: Any
    • Service: HTTP (This automatically sets Port 80)
    • Action: Allow
    • Priority: 100 (Lower numbers have higher priority)
    • Name: Allow-HTTP
  7. Click Add.

Exercise 3: Associate the NSG to the Subnet

An NSG does absolutely nothing until it is attached to a Subnet or a specific Virtual Machine's Network Interface (NIC). We will apply our new NSG to the Frontend subnet.

  1. While still inside the Frontend-NSG resource menu, look under Settings and click Subnets.
  2. Click + Associate at the top.
  3. In the dropdown for Virtual Network, select Core-VNet.
  4. In the dropdown for Subnet, select Frontend.
  5. Click OK.
Verification: You have successfully secured the network layer! Any Virtual Machine you deploy into the Frontend subnet will now automatically be protected by this NSG, allowing HTTP traffic in while blocking other unapproved inbound internet traffic (like RDP or SSH).

Exercise 4: Host your Domain on Azure DNS

Azure DNS allows you to host your DNS domain in Azure to manage your DNS records using the same credentials, APIs, tools, and billing as your other Azure services.

  1. In the Azure portal search bar, type DNS zones and select it.
  2. Click + Create.
  3. Select your Subscription and the Resource Group you have been using.
  4. Enter a name for the zone, such as contosolab.com (you can use a dummy domain name for this test).
  5. Click Review + create, then click Create.
  6. Once deployed, click Go to resource.
  7. Click + Record set at the top menu to create a new DNS mapping.
  8. Leave the Name field blank (which resolves the root domain) or type www. Ensure the Type is set to A.
  9. In the IP address field, enter a dummy IP address (e.g., 10.1.2.3) or the public IP of a real VM if you have one.
  10. Click OK to save the record.
Verification: You have successfully hosted a DNS zone and created an A record in Azure! To use this in production, you would copy the four Azure Name Servers (NS1, NS2, etc.) listed on the Overview page and paste them into your domain registrar's settings (like GoDaddy or Namecheap).
