Secure Resources with Azure RBAC

Learn how to manage access using Role-Based Access Control assignments, scopes, and auditing logs.

Concept Refresher: A Role Assignment consists of three parts: a Security Principal (Who needs access?), a Role Definition (What can they do?), and a Scope (Where can they do it?).

Exercise 1: View Existing Access

Before granting new permissions, an administrator must know how to audit existing permissions at a specific scope.

  1. Sign in to the Azure portal.
  2. In the search bar, search for and select Resource groups.
  3. Select any existing resource group (or create a quick temporary one).
  4. In the left-hand navigation menu, click on Access control (IAM). IAM stands for Identity and Access Management.
  5. Click on the Check access tab at the top.
  6. Search for your own user account name in the search bar and select it.
  7. Review your current assignments. You will likely see "Owner" or "Contributor" inherited from the Subscription scope.

Exercise 2: Grant Access using RBAC

In this exercise, we will follow the principle of least privilege. We will grant a user permission to manage Virtual Machines, but only within one specific Resource Group.

  1. Still inside your chosen Resource Group, navigate back to the Access control (IAM) page.
  2. At the top of the page, click the + Add button, then select Add role assignment.
  3. On the Role tab, use the search bar to find Virtual Machine Contributor. Select it and click Next.
  4. On the Members tab, ensure "Assign access to" is set to User, group, or service principal.
  5. Click + Select members. Search for the user we created in the previous module (e.g., "Chris Green") or any other user in your tenant. Select them.
  6. Click Review + assign at the bottom, and then click it again to finalize.
Verification: Go to the Role assignments tab in IAM. You should now see "Chris Green" listed with the "Virtual Machine Contributor" role. Notice the Scope column says "This resource" (meaning the Resource Group level).

Exercise 3: View Activity Logs for RBAC Changes

For security and compliance audits, Azure tracks exactly who granted access and when it happened.

  1. In the same Resource Group, scroll down the left-hand menu and click on Activity log.
  2. Change the Timespan filter to Last 6 hours if it isn't already.
  3. Look for an Operation Name called Create role assignment. It might take a minute or two to appear (click Refresh at the top if needed).
  4. Click on the event to view the summary.
  5. Click the JSON tab. This raw data contains the exact details of the API call, including the caller's IP address, the role ID, and the principal ID. This is the exact data that gets sent to security tools like Microsoft Sentinel!
