Module 1 Deep Dive

Microsoft Entra ID (Azure AD)

Master user identities, group management, and administrative units.

Understanding the Identity Perimeter

In modern cloud architecture, the network firewall is no longer the primary security boundary. Identity is the new perimeter. Entra ID is a multi-tenant, cloud-based directory and identity management service.

Exam Tip:

Know the difference between Windows Server Active Directory (uses Kerberos/NTLM, requires domain controllers) and Entra ID (uses HTTP/HTTPS protocols like SAML, WS-Federation, and OAuth 2.0).

User and Group Management Concepts

Managing access at scale requires understanding the difference between user assignment types:

  • Assigned (Direct): You manually add a specific user to a group or assign a role directly. This is hard to maintain at scale.
  • Dynamic User: Users are automatically added or removed from a group based on their user attributes (e.g., user.department -eq "Sales").
  • Dynamic Device: Devices are grouped automatically based on OS, version, or ownership (Requires Intune/Entra ID P1).

Advanced Lab: Bulk User Import

In the real world, you rarely create users one by one. You will use a CSV template to bulk create identities.

  1. In the Azure Portal, search for Microsoft Entra ID.
  2. Navigate to Users > All users.
  3. At the top menu, click Bulk operations > Bulk create.
  4. Download the provided CSV template.
  5. Fill out the CSV with user Principal Names (e.g., user1@yourtenant.onmicrosoft.com), Display Names, and initial Passwords.
  6. Upload the file and click Submit. Wait for the notification indicating success.
